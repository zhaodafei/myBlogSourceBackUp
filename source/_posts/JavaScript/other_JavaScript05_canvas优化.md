Canvas缓存策略,核心优化思路
1. 分层缓存：将绘制内容按 “更新频率” 拆分
2. 视口裁剪：只绘制 “看得见” 的标注点
3. 脏区更新：仅重绘 “变化的区域”
4. 批量绘制：减少 Canvas API 调用次数
5. 渲染调度：降低无效绘制频率

<!-- more -->

### 总结
1. **分层缓存**是核心：拆分静态 / 动态内容，静态内容只绘一次、复用缓存，从根源减少绘制量；
2. **视口裁剪 + 脏区更新**是关键：只处理 “可视的、变化的” 区域，过滤无效绘制；
3. **批量绘制**是补充：减少 Canvas API 调用开销，提升单次绘制效率。
