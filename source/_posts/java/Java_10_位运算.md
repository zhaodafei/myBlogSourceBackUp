Java入门基础-位运算

<!-- more -->

### 基础理解

<font color="#ff6b81">`64`和`1 << 6`</font>数值结果完全相同（均为 64），但二者的语义、底层逻辑、适用场景有本质区别，核心差异源于 “直接常量” 与 “位运算表达式” 的本质不同。下面从多个维度详细解析：

- 64：这是一个直接的 字面量。代码看到的就是数字 64。

- 1 << 6：这是一个 位运算表达式。

  > - `<<` 是左移运算符。
  > - `1 << 6` 的意思是：将数字 1（二进制 `0b00000001`）的二进制位向左移动 6 位，右边空出的位用 0 填充。
  > - 结果是 `0b01000000`，也就是十进制的 64。

  **结论**：在程序编译或运行后，两者的 **数值是完全相等的**。


### 主要区别

#### 1、可读性与意图表达

> **使用 `64`**：
> 当这个数字本身的意义就是“六十四”这个数量时。例如，定义一个数组长度、一个超时时间、一个普通的数学常量。
>
> ```js
> MAX_BUFFER_SIZE = 64
> timeout_seconds = 64
> ```

> **使用 `1 << 6`**：
> 当这个数字代表一个 **位标志** 时，它能清晰地表达“这是第 6 位（从 0 开始数）被设置为 1”。这在处理权限、状态码、选项等“位掩码”时非常有用。
>
> ```js
> # 定义一组权限
> PERMISSION_READ = 1 << 0   # 0b00000001 (第0位)
> PERMISSION_WRITE = 1 << 1  # 0b00000010 (第1位)
> PERMISSION_DELETE = 1 << 2 # 0b00000100 (第2位)
> # ...
> PERMISSION_ADMIN = 1 << 6  # 0b01000000 (第6位) —— 这里清晰地表明这是第6个权限位
> 
> # 给用户添加管理员权限
> user_permissions |= PERMISSION_ADMIN
> ```
>
> 如果你在这里写 PERMISSION_ADMIN = 64，其他开发者需要心算一下“64 是 2 的几次方？”，才能知道它对应的是第几位。而 1 << 6 则一目了然。

#### 2、可维护性

当需要调整位的位置时，`1 << n` 的形式更有优势。

假设你需要将管理员权限从第 6 位改为第 7 位：

- 使用 `1 << 6`：只需改为 `1 << 7`。
- 使用 `64`：你需要重新计算 `1 << 7` 的值是 128，然后改为 `128`。这更容易出错，尤其是在一个有很多位标志的枚举中。

#### 3、性能

对于现代编译器（如 C/C++, Java, Rust 等）和解释器（如 Python 的高版本），`1 << 6` 这种常量表达式在 **编译时** 就会被直接计算成 `64`。所以最终生成的机器码或字节码是完全一样的，**没有任何性能差异**。

### 总结

何时使用哪个?

- **当你处理的是位操作、标志位、掩码时，强烈推荐使用 `1 << n`**。它传达了“这是一个位索引”的重要信息。
- **当你处理的只是一个普通的整数数值时，直接使用 `64`** 更简洁明了。

简单来说：**`64` 告诉你“是多少”，而 `1 << 6` 告诉你“是第几位”。**
